Lambda Expression
-----------------

Es otra forma de escribir la implementación de una interfaz, menos verbosa y más leíble. 

Se pueden usar en declaraciones de tipos que sean Functional Interface.


Method Reference
----------------

En métodos que reciben este tipo de interfaces se pudiera pasar una implementaión como clase anónima (más verbosa, boilerplate code),
o una expresión lambda o para simplificar más una referencia a un método.

Se puede usar como referencia a un método, en la forma Class::instanceMethod, un metodo de instancia de la misma clase que se recibe
como parámetro en el método de la functional interface, siempre que retornen el mismo tipo. 

Si se usa esta forma le estamos diciendo a Java que asumimos que se va a invocar sobre el objeto que se pase como argumento el método
de instancia que hemos declarado como referencia.

(obj, args) -> obj.instanceMethod(args)
